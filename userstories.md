# Inyon User Stories

## Implemented

- As a new user, I can see onboarding entry with `Get Started` and login path selection.
- As a new user, I can provide birth date, city, optional birth time, and personal anchors.
- As a new user, I can select an insight tone preference (Calm or Sharp) during onboarding.
- As a user, onboarding context (including tone preference) persists to Firestore.
- As a signed-in user, I am routed into the main app; signed-out users remain in onboarding.
- As a new/returning user, I can sign up and log in with email/password.
- As a new/returning user, I can sign in with Apple.
- As a new/returning user, I can sign in with Google.
- As a user, onboarding save uses returned account UID to avoid auth-listener race.
- As a user, anonymous onboarding migration is transactional:
  - existing authenticated fields win on conflict
  - anonymous onboarding doc is deleted only on successful transaction commit
  - migration errors propagate and block completion path
- As a user, I receive a personalized daily reflection generated by a Cloud Function (OpenAI gpt-4o-mini) using my Saju birth context, personal anchors, and day's heavenly stem/earthly branch.
- As a user, I see the Environment×You section on the Home screen showing today's element dynamic and dynamicText.
- As a user, my daily insight is cached in Firestore and served from cache on repeat visits, keyed by date, timezone, and tone preference.
- As a user, I can edit all profile fields in the You tab and persist changes to Firestore.
- As a user, I can change my insight tone preference in the You tab; the next insight fetch reflects the new tone.
- As a user, I can share today's reflection as a PNG card via the native iOS share sheet (ambient share icon, no intrusive CTA).
- As a signed-in user, I can log out from the You tab and return to onboarding.
- As a user, I can view the Guide (Lens) tab explaining what Saju is, including tappable element detail sheets.
- As a user, I can enable daily insight notifications with a configurable delivery time.
- As a team, Firebase Analytics is wired for key events.
- As a team, pre-push blocks push unless unit and UI/E2E test suites pass.
- As a team, pre-push simulator selection is exact-name and robust to `iPhone 16e` vs `iPhone 16`.
- As a team, a `/review` slash command runs an AI code review before any PR is opened.

## Partially Implemented

- As a user, I receive clear auth failure feedback and can retry.
  - Implemented for email and social auth flows with visible error + retry action.
  - Edge cases for all provider failure states (e.g. network loss mid-auth) not fully tested on device.
- As a user, notification scheduling is timezone-aware and delivers at the user's chosen time.
  - UI (toggle + time picker) is fully wired in You tab with `UNUserNotificationCenter`.
  - End-to-end delivery timing needs verification on a physical device.
- As a team, automated tests cover core paths.
  - 51 unit tests cover onboarding, auth, YouViewModel (load/save/tone/notifications), HomeViewModel, and social auth.
  - Broader integration coverage against live backend and full UI flows is still limited.

## Not Implemented

- As a team, analytics is fully production-wired beyond existing event logging (funnels, retention dashboards not configured).

## How To Verify Before Push

```bash
bash scripts/prepush_hook_tests.sh
bash -n .githooks/pre-push

SIM_DEST="$(bash scripts/select_simulator.sh --destination)"
xcodebuild test -project Inyon.xcodeproj -scheme Inyon -destination "$SIM_DEST" -only-testing:InyonTests
xcodebuild test -project Inyon.xcodeproj -scheme Inyon -destination "$SIM_DEST" -only-testing:InyonUITests
```

## Deploy Checklist (before shipping a build)

- `firebase deploy --only functions` — required after any Cloud Function change (tone preference prompt, new fields)
- `firebase deploy --only firestore:rules` — required after any Firestore rules change
- Rotate OpenAI API key if needed: `firebase functions:secrets:set OPENAI_API_KEY`
- Verify MapKit city search on physical device (no location permission crash)
- Test social auth (Apple + Google) on physical device — simulators have known limitations
